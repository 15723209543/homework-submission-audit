# Homework Submission Audit Skill

一个用于辅助检查班级作业提交情况的 AI Skill。

本 Skill 面向班级作业、课程实验、课程设计、小组报告等批量提交场景，可根据班级名单、作业要求及模板、学生提交文件，对提交情况和文档问题进行初步整理，并生成统一格式的 `.output` 与 `.errcode` 文件。

---

## 1. Skill 功能概述

本 Skill 用于辅助完成以下任务：

1. 检索班级名单，判断每一位学生是否提交作业。
2. 检查提交文件中的姓名、学号等个人信息是否与班级名单匹配。
3. 检查提交文件格式是否符合题目要求及模板。
4. 检查提交内容是否完成作业要求。
5. 分析提交内容是否存在较高 AI 痕迹或疑似抄袭风险。
6. 检查提交文件是否存在明显异常，例如无法打开、文件损坏、编码异常、内容为空、文件大小异常等。
7. 根据用户额外提出的特殊要求进行补充检查，并使用 6 开头错误码记录。
8. 输出两个统一格式的结果文件：`.output` 和 `.errcode`。

---

## 2. 适用场景

本 Skill 适合以下场景：

- 班级作业批量检查
- 实验报告提交情况核对
- 课程设计报告初步筛查
- 小组作业、个人作业提交整理
- 助教或班委对提交文件进行初步汇总
- 需要把检查结果统一输出为文本文件的场景

本 Skill 不适合作为最终评分依据，也不适合替代教师、助教或课程负责人的人工复核。

---

## 3. 触发方式

当用户提出类似以下需求时，可以使用本 Skill：

```text
请检查我们班作业提交情况。
请根据名单和提交文件生成 output 和 errcode。
请检查每个人是否提交，没提交返回 NF。
请检查作业格式、个人信息、AI 痕迹和文件异常。
请生成 GBK 编码的 .output 和 .errcode 文件。
```

## 4. 输入文件要求

使用本 Skill 时，通常需要提供以下三类输入。

### 4.1 班级名单

班级名单应至少包含：

- 姓名
- 学号

建议使用 `.xlsx`、`.xls`、`.csv`、`.txt` 等格式。

### 4.2 作业题目要求及模板

作业要求及模板应尽量包含：

- 作业题目
- 提交要求
- 文件命名要求
- 文档格式要求
- 必填内容
- 模板文件
- 其他特殊检查要求

### 4.3 班级提交作业内容

班级提交内容建议统一打包为：

- `.zip`
- `.7z`
- `.rar`

压缩包中可以包含：

- Word 文档
- PDF 文档
- 表格文件
- 文本文件
- 图片
- 源代码
- 其他本次作业允许提交的文件

---

## 5. 输出文件要求

本 Skill 输出两个文本文件。

输出文件名不固定，可根据任务名称自定义，例如：

```text
homework.output
homework.errcode
experiment1.output
experiment1.errcode
class_report.output
class_report.errcode
```

建议直接使用 `.output` 与 `.errcode` 作为后缀，便于区分检查结果文件和错误码说明文件。

---

## 6. `.output` 文件说明

`.output` 文件用于记录每一位学生的提交情况。

### 6.1 编码要求

`.output` 文件使用 GBK 编码输出，确保 Windows 记事本可以正常打开。

### 6.2 文件格式

文件开头为输出说明，说明部分每一行以 `#` 开头。

随后每一位学生一行，格式为：

```text
姓名    学号    提交情况
```

各列按照最大字符宽度进行制表，保证整体对齐。

### 6.3 提交情况说明

| 返回值 | 含义 |
|---|---|
| `SUCC` | 已提交，且未发现明显问题 |
| `NF` | 名单中有该学生，但未检索到提交文件 |
| 三位错误码 | 已提交，但存在对应问题 |
| 多个错误码 | 同一学生存在多个问题，错误码之间用空格分隔 |

示例：

```text
# 文件名称：自定义文件名.output
# 文件编码：GBK
# 文件说明：本文件用于输出班级作业提交检查结果。
# 输出格式：姓名 + 学号 + 提交情况
# 提交情况说明：SUCC 表示检查通过；NF 表示未提交；其他三位数字表示对应错误码。
# 如果一名学生存在多个问题，错误码之间使用空格分隔。
# 如果出现班级名单中不存在的提交者，应输出在文件最后。
# 重要说明：除是否提交外，本文件中的其他检查结果均为辅助判断结果，仅供参考，不作为最终认定依据。
# 免责声明：最终结果应以教师、助教、课程负责人或人工复核结果为准，文件生成者不对使用本结果产生的后果负责。
# 联系方式：15723209543@163.com

姓名        学号            提交情况
张三        2025000001      SUCC
李四        2025000002      NF
王五        2025000003      101 203
赵六        2025000004      301 401 601
名单外提交  未知            104
```

## 7. `.errcode` 文件说明

`.errcode` 文件用于记录错误码分类、错误码含义及修改建议。

### 7.1 编码要求

`.errcode` 文件使用 GBK 编码输出，确保 Windows 记事本可以正常打开。

### 7.2 文件格式

文件开头为输出说明，说明部分每一行以 `#` 开头。

随后每一个错误码一行，格式为：

```text
错误码分类    错误码    应该如何修改
```

各列按照最大字符宽度进行制表，保证整体对齐。

示例：

```text
# 文件名称：自定义文件名.errcode
# 文件编码：GBK
# 文件说明：本文件用于说明作业检查过程中出现的错误码含义及修改建议。
# 输出格式：错误码分类 + 错误码 + 应该如何修改
# 错误码规则：1 开头表示个人信息错误；2 开头表示格式错误；3 开头表示任务完成错误；4 开头表示 AI 或抄袭风险；5 开头表示文件异常；6 开头表示其他任务需求错误。
# 特别说明：错误码为现场申请，不提前固定全部情况；检查过程中发现新的具体问题时，应申请新的错误码，并在本文件中登记。
# 重要说明：本文件中的错误码解释和修改建议均为辅助判断结果，仅供参考，不作为最终成绩评定或最终违规认定依据。
# 免责声明：最终结果应以教师、助教、课程负责人或人工复核结果为准，文件生成者不对使用本结果产生的后果负责。
# 联系方式：15723209543@163.com

错误码分类        错误码      应该如何修改
个人信息错误      101        修改提交文件中的姓名，确保与班级名单中的标准姓名一致。
格式错误          201        按照作业模板重新调整文档结构。
任务完成错误      301        补全缺少的必做题目或必交内容。
AI或抄袭风险      401        降低机械化表达，补充个人理解、真实过程和具体细节。
文件异常          501        重新提交可以正常打开的文件。
其他任务需求错误  601        按照本次额外要求补充指定材料。
```

## 8. 错误码规则

错误码统一使用三位数字表示。

错误码不是固定死的，实际检查时按照现场发现的问题动态申请和登记。本 Skill 只规定错误码的大类开头，不提前限制具体错误码数量。

### 8.1 `1xx`：个人信息错误

用于表示学生提交文件中的个人信息与班级名单不一致。

可能包括但不限于：

- 姓名不匹配
- 学号不匹配
- 姓名缺失
- 学号缺失
- 提交文件中个人信息无法识别
- 文件名信息与正文信息冲突

### 8.2 `2xx`：格式错误

用于表示学生提交文件格式与题目要求或模板不一致。

可能包括但不限于：

- 文件格式不符合要求
- 文档结构与模板不一致
- 标题、章节或排版格式缺失
- 未按要求命名文件
- 页面、字体、字号、段落格式明显不符合模板
- 表格、图片、附件位置不符合要求

### 8.3 `3xx`：任务完成错误

用于表示学生提交内容没有完整完成题目要求。

可能包括但不限于：

- 缺少必做题目
- 缺少必要分析内容
- 缺少实验结果或运行截图
- 缺少测试数据
- 内容明显与题目要求不对应
- 只完成部分任务
- 关键结论缺失

### 8.4 `4xx`：AI 或抄袭风险

用于表示文档存在较高 AI 痕迹或疑似抄袭情况。

可能包括但不限于：

- AI 生成痕迹较高
- 与其他学生提交内容高度相似
- 存在明显复制粘贴痕迹
- 语言风格与正常学生作业明显不符
- 多份文件结构、语句、错误高度一致
- 个人过程描述明显空泛

### 8.5 `5xx`：文件异常

用于表示提交文件本身存在异常。

可能包括但不限于：

- 文件无法打开
- 文件大小异常
- 文件编码异常
- 文件内容为空或严重缺失
- 提交文件损坏
- 文件后缀与实际内容不一致
- 压缩包结构异常
- 重复提交多个版本且无法判断最终版

### 8.6 `6xx`：其他任务需求错误

用于表示本次检查任务中额外提出、但不属于前五类的特殊检查要求。

该类错误码根据每一次具体任务现场申请，适合记录临时增加的检查项目或特殊规则。

可能包括但不限于：

- 未按本次额外说明提交指定材料
- 未满足临时增加的统计、排序、分类或汇总要求
- 未按额外要求提供截图、附件、说明文件或补充材料
- 未按照特殊命名规则、特殊文件夹结构或特殊提交顺序整理
- 本次任务特别要求检查的其他问题

## 9. 错误码现场申请规则

如果检查时发现某一类新的具体问题，应按照以下规则现场申请错误码：

1. 先判断问题属于哪一大类。
2. 根据大类选择对应开头数字。
3. 在该大类下选择尚未使用的三位错误码。
4. 将错误码、错误码分类、修改建议写入 `.errcode` 文件。
5. 在 `.output` 文件中对应学生的提交情况位置写入该错误码。
6. 如果同一个学生存在多个问题，则错误码之间用空格分隔。

示例：

```text
个人信息错误      105      修改正文首页学号，确保正文个人信息与班级名单一致。
其他任务需求错误  601      补交本次任务额外要求的附件说明文件。
```

## 10. Skill 工作流程

建议按照以下流程执行：

1. 读取班级名单，提取标准姓名和标准学号。
2. 解压并整理班级提交文件。
3. 根据文件名、正文首页、文档内容等信息匹配提交学生。
4. 对名单中每一位学生判断是否提交。
5. 对已提交学生检查个人信息。
6. 对已提交文件检查格式和模板一致性。
7. 对已提交内容检查任务完成情况。
8. 对文档进行 AI 痕迹和相似度风险分析。
9. 对文件本身进行异常检查。
10. 根据用户额外要求进行 6 开头错误码检查。
11. 生成 `.output` 文件。
12. 生成 `.errcode` 文件。
13. 确保两个输出文件均为 GBK 编码。

## 11. 使用建议

为了提高检查准确率，建议用户提供：

- 完整班级名单
- 明确的作业要求
- 原始模板文件
- 完整提交压缩包
- 文件命名规则
- 是否允许补交、重复提交、多人一组提交等特殊说明
- 是否需要检查代码、截图、附件、运行结果等额外内容

---

## 12. 注意事项

1. 本 Skill 的检查结果主要用于初筛，不建议直接作为最终成绩依据。
2. AI 痕迹和抄袭风险只能作为风险提示，不能作为最终违规认定。
3. 格式错误和任务完成错误可能受模板复杂度、文件解析效果影响，需要人工复核。
4. 如果提交文件损坏、加密、无法读取，可能只能给出文件异常类错误码。
5. 如果名单信息不完整，可能影响是否提交和个人信息匹配判断。
6. 如果存在重名、多人一组、代交、重复提交等情况，应结合人工确认。

最终认定结果应以教师、助教、课程负责人或人工复核结果为准。
